## AI エージェントの使用手順

### 前提

- 環境構築手順がすべて完了し、ECS Service が起動済み（`正常` ステータス）であること
- 利用したい Slack チャンネルに本製品の Slack App が招待済みであること

### 質問の送り方

1. Tracis を利用したい Slack チャンネルを開く
2. メッセージ欄で `@<Slack App の名前>` を入力し、続けて調べたい内容を日本語で入力して送信する
3. しばらく待つと、同じスレッド内に Tracis から回答メッセージが投稿される

> Tracis はメンション（`app_mention`）されたメッセージのみを処理します。メンションを付けずに発言した場合は応答しません。

### 質問例

Tracis は CloudWatch Logs および MySQL のメタ情報を参照して回答します。質問はできるだけ具体的に、調査範囲（時間帯・対象システムなど）を含めて記述すると精度が向上します。

| 例                 | 質問内容                                                                      |
| :----------------- | :---------------------------------------------------------------------------- |
| ログ調査           | `@Tracis 昨日 10:00〜12:00 の間で order-service にエラーログは出ていますか？` |
| 識別子追跡         | `@Tracis request_id=abc-123 のリクエストがどこで失敗したか調べてください`     |
| ユーザー単位の追跡 | `@Tracis user_id=42 が直近1時間で行った操作と、その結果を教えてください`      |
| 異常検知           | `@Tracis 直近30分で 500 系エラーが急増しているサービスはありますか？`         |

### 回答までの目安

- 通常、1 つの質問に対する回答までの所要時間は **数十秒〜数分** 程度です
- 調査範囲が広い場合や試行回数が増えると、上記より時間がかかることがあります
- タイムアウト時間の上限を超えた場合は、回答が返らないまま処理が打ち切られます。範囲を絞って再度質問してください

### 使用時の注意事項

- **回答は実データに基づきます**：Tracis は CloudWatch Logs と MySQL の実データを参照して回答します。ログに残っていない情報や、ナレッジベースに登録されていない知識については回答できません。
- **ナレッジベースの内容に依存します**：質問対象のロググループやテーブル構造などのメタ情報は [ナレッジベース](/docs/create_environment/05_prepare_knowledgebases/README.md) に登録されている内容を参照します。期待した回答が得られない場合、ナレッジベースの内容を見直してください。
- **MySQL は識別子経由でのみ参照されます**：オーケストレーターはまず CloudWatch Logs を調査し、ログから識別子（`request_id`、`user_id`、`order_id` など）を抽出できた場合のみ MySQL を検索します。識別子の手がかりを含む質問にすると、より深い調査につながります。
- **クエリ試行回数には上限があります**：同一質問に対するクエリ試行は最大 3 回までです。3 回試行しても有効な結果が得られない場合、その旨を報告して調査を終了します。
- **個人情報・機微情報の扱いに注意してください**：ログや DB の内容がそのまま Slack に投稿されるため、機微情報を含む可能性があるロググループ／テーブルをナレッジベースに登録する際は事前にご確認ください。
- **同時実行に関する制約**：リクエストは SQS（FIFO キュー）経由で処理されるため、同一スレッド内で連続して質問を送信した場合は順次処理されます。回答が返る前に追加で質問を送ると、回答までの待ち時間が長くなる場合があります。
- **エージェントが停止しているときは応答しません**：費用削減等のため ECS Service を停止している場合、メンションを送っても回答は返りません。停止中の起動方法は [起動・停止手順](/docs/create_environment/07_agent_start_stop/README.md) を参照してください。
- **利用に応じてコストが発生します**：質問ごとに Bedrock（LLM）のトークン使用コストが発生します。また、ECS Service が起動している間は Fargate の稼働コストが継続して発生します。費用を抑えるため、利用しない時間帯はエージェントを停止することを推奨します（[起動・停止手順](/docs/create_environment/07_agent_start_stop/README.md)）。なお、CloudFormation スタックの `UseFargateSpot` パラメータを有効にすることで、Fargate Spot を利用して稼働コストを抑えることも可能です。ただし、エージェントの処理が中断される可能性があります。
- **詳細な参考コストについて**：想定される AWS 利用料金および AI 利用料金の参考値については [参考コスト](/docs/COSTS.md) を参照してください。
- **モデルの最大トークン数は変更できます**：Bedrock モデルが一度に扱うことができるトークン数の上限は、CloudFormation スタックの `BedrockModelMaxTokens` パラメータで設定されています。この上限を超えるトークンを処理しようとした場合、エージェントの処理が失敗する可能性があります。必要に応じてスタックのパラメータを更新することでこの上限を変更できます。ただし、トークン上限を増やすほど Bedrock のトークン使用コストも増加します。
